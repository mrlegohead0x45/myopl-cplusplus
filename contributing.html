Contribute¶

If you want to contribute to this project, great! We would really appreciate your contribution

Contributing¶

Get started by creating a fork of the GitHub repository and cloning it locally, preferably with the --recurse-submodules flag to download vcpkg, which enables you to skip the git submodule commands lower down.

Advice¶

You may want to add build/vcpkg_installed/triplet/include and include to your IDE’s include search path, as this is where installed header files and our headers are respectively. triplet should be your vcpkg target triplet (see Triplets). Or you could just look.

For example in my .vscode/c_cpp_properties.json I have

{
    "configurations": [
        {
             ...
            "includePath": [
                "${default}",
                "${workspaceFolder}/include",
                "${workspaceFolder}/build/vcpkg_installed/x64-mingw-dynamic/include"
            ]
            ...
        }
    ]
}

Submitting your changes¶

If you think you have made an improvement, no matter how small, please submit your changes as a pull request.

Please run clang-format on your code before you submit your pull request, like so $ clang-format -i --style=file changed files. Please also make sure your code compiles, following the Compiling instructions.

Compiling¶

Prerequisites¶

  1. Have CMake

  2. Have Git

  3. Have a C++ compiler

Any text that starts with a $ should be run from the command line (without the $)

vcpkg¶

You will need to run $ git submodule init and $ git submodule update in order to download vcpkg. Then run $ ./vcpkg/bootstrap-vcpkg.sh (or $ .\vcpkg\bootstrap-vcpkg.bat on Windows) to set it up.

Compiling¶

Run CMake to generate build files (known as configuring), specifying -DVCPKG_TARGET_TRIPLET=triplet if necessary (see Triplets) and -G generator as well if necessary (see CMake generators)
$ cmake -B build -S .
Then you can run those build files to actually compile the code
$ cmake --build build
You might want to pass the -j jobs flag to speed up the build by running multiple processes. To find a suitable value, on Linux you can run $ nproc in a terminal. On Windows, you can go to Start > Task Manager > More details > Performance > CPU > Logical processors.

Triplets¶

If you are cross-compiling (compiling with a compiler that is not native to your system), you may need to specify the target triplet. For example, on my machine with MinGW installed I need to pass -DVCPKG_TARGET_TRIPLET=x64-mingw-dynamic to CMake which tells vcpkg to build the dependencies for use with MinGW.

Docs¶

If your changes include modifying the documentation, you will want to build the docs to check that they do actually build and that the build looks how you intended it to. To do this you will need Python and Sphinx. To build the docs, cd to the docs directory and on Windows run $ .\make.bat html and on *nix run $ make html, which will build the docs and place them in docs/build/html with index.html as the homepage.
